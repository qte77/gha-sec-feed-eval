### Added

- **Phase 2b: core eval library + reusable workflow (#3).** All 10
  modules under `src/gha_sec_feed_eval/` shipped via strict TDD
  (Red ‚Üí Green per module, 20 commits + tests). Coverage gate flipped
  from `--cov-fail-under=0` to `--cov-fail-under=70`; actual coverage
  on `src/` is 100% (446/446 stmts) over 142 non-trivial tests.

- **`models.py`** ‚Äî strict pydantic v2 contracts for C1 (`FeedRow`),
  C2 (`PriorityRow`), and `priority-meta.json` (`Meta`). Schema pinned
  at `schema_version: "1.0.0"`. Per-field strict typing
  (`StrictStr`/`StrictFloat`/`StrictBool`) blocks silent coercion;
  `AwareDatetime` rejects naive `published` timestamps.
- **`scoring.py`** ‚Äî locked 0-10 priority formula (KEV +2 /
  active-exploit +2 / EPSS√ó2 / recency / CVSS‚â•9 +2), clamped to 10
  and rounded to 2 decimals. `has_active_exploit(row)` heuristic
  matches `source=cisa-kev`, exploit-db / metasploit-framework /
  packetstormsecurity ref hosts, and `/poc/` / `/exploit` path
  signals. `priority_category()` thresholds at 8.0 / 5.0.
- **`filter.py`** ‚Äî loads `categories/default.yaml` into a strict
  `CategoriesConfig`; `matched_categories(row, config)` returns the
  ecosystem slugs whose keywords appear in the row's refs
  (case-insensitive substring, end-`*` wildcard support).
- **`loader.py`** ‚Äî `parse_feed(text)` parses JSONL with line-number
  context in errors; `SchemaVersionError(LoaderError)` raised on
  schema drift (brief-mandated stop-and-ask). Blank/whitespace lines
  skipped.
- **`http_client.py`** ‚Äî single egress chokepoint with brief-mandated
  5-host allowlist ([raw.githubusercontent.com](https://raw.githubusercontent.com/),
  [api.github.com](https://api.github.com/), [attack.mitre.org](https://attack.mitre.org/),
  [d3fend.mitre.org](https://d3fend.mitre.org/), [api.first.org](https://api.first.org/)),
  https-only scheme guard, and `GSFE_OFFLINE` short-circuit. RFC-3986
  case-insensitive host comparison.
- **`enrich/attack.py`** + `vendor/attack-stix.json` ‚Äî CVE ‚Üí ATT&CK
  technique-ID lookup against a vendored *subset* (10 high-profile
  CVEs seeded; Log4Shell / xz / runc / spring4shell / etc.) per the
  Phase 2b decisions memo. NOT the full STIX bundle.
- **`enrich/d3fend.py`** + `vendor/d3fend-mappings.json` ‚Äî ATT&CK
  technique ‚Üí D3FEND countermeasure-ID lookup against a vendored
  subset; coverage-pinned to every technique referenced by
  `vendor/attack-stix.json`.
- **`enrich/epss.py`** ‚Äî `resolve_epss(row, http_get=...)` follows the
  C1.epss ‚Üí offline ‚Üí fetch precedence. Live `api.first.org` fetch
  test gated behind [`@pytest.mark.network`](https://docs.pytest.org/en/stable/how-to/mark.html)
  per the Phase 2b decisions.
- **`writer.py`** ‚Äî `write_outputs(rows, output_dir, ...)` emits
  `priority.jsonl` + `priority-meta.json` + `REPORT.md`. REPORT.md
  carries frontmatter for MD041 + three bucket sections; polish lands
  in Phase 2c (#4).
- **`config.py`** ‚Äî `AppSettings` (pydantic-settings) with `GSFE_`
  env prefix + frozen.
- **`cli.py`** ‚Äî `run(settings, http_get, input_file=None)`
  orchestrates the pipeline; `main(argv)` is the argparse entry.
  `--feed-url` / `--input-file` are mutually exclusive; the latter
  enables local-fixture runs (`make smoke`) without a live producer.

- **`.github/workflows/eval.yaml`** ‚Äî reusable
  (`on: workflow_call`) consumer-facing workflow with the load-bearing
  `eval_ref: required: true` self-checkout pattern per
  [`qte77/gha-rxiv-paper-eval`](https://github.com/qte77/gha-rxiv-paper-eval)
  v0.2.1. Outputs `relevant_count` + `priority_artifact`.
- **`.github/workflows/eval-dispatch.yaml`** ‚Äî manual-trigger wrapper
  with `concurrency:` block.

- **`tests/fixtures/feed-min.jsonl`** ‚Äî 5-row realistic C1 fixture
  covering Log4Shell, xz, runc, spring4shell, and a synthetic future
  CVE. Drives both unit tests and `make smoke`.
